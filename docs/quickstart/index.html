Open OnDemand
Quickstart Guide

A web-based portal that gives you browser access to Trillium's computing resources — no software installation required. Run Jupyter, RStudio, VSCode, manage files, submit jobs, and more.

🖥️ Interactive Apps 📁 File Management ⚙️ Job Submission 💻 Web Terminal 📊 Job Monitoring
Portal URL https://ondemand.scinet.utoronto.ca
ℹ️
This page covers the Open OnDemand service attached to Trillium. For general information on Open OnDemand instances across all Alliance clusters, see the Open OnDemand wiki page.
🌐

Web-based Access

Use any modern browser — no SSH client or VPN needed.

⚡

Interactive Computing

Run Jupyter, RStudio, VSCode and more with dedicated CPU/GPU resources.

📂

File Browser

Upload, download, create and edit files up to 10 GB directly in your browser.

🎯

Job Scheduler

Submit and monitor Slurm batch jobs with the Open Composer interface.

Logging into the Portal

Open a web browser and navigate to the portal URL below. You will be prompted for your Alliance username and password, followed by multi-factor authentication via Duo or Yubikey.

Portal URL https://ondemand.scinet.utoronto.ca

Once logged in, you will land on the Open OnDemand dashboard where all tools and applications are accessible from the top menu bar.

📦 Uploading Large Files — Use Globus

The file upload limit in the browser is 10 GB. For larger transfers or unreliable connections, use Globus.

💡
A Globus button is available in the top-right of the file browser. Clicking it opens Globus at the same path you were browsing in OOD — log in with your Alliance credentials.

Job Submission

Open OnDemand uses Open Composer — a suite of pre-built Slurm job templates — to submit batch jobs to Trillium. This is ideal when you need more resources than interactive sessions offer (up to 192 cores and 755 GB RAM on a full compute node).

📍
Access Open Composer via the Jobs drop-down → Open Composer, or click one of the quick-launch shortcuts: MPI Slurm Job, OpenMP Slurm Job, or Hybrid MPI/OpenMP Slurm Job.

Job Monitoring

Active Jobs View

Navigate to Jobs → Active Jobs for a live overview of your queue.

  • Use the Filter box (top-right) to search jobs
  • Click column headers to sort (e.g. by status)
  • Click > beside a job to see start/end time, node list, and charged account
  • Use the dropdown to switch between My Jobs and All Jobs

For detailed performance statistics, visit the mySciNet portal.

Open Composer History Tab

Navigate to Jobs → Open Composer → History to see all jobs submitted through Open Composer.

Column clickAction

ApplicationOpens the job script template editor

Script NameDisplays the submitted job script

To re-run or modify a previous job: click the script under Script Name, then click Load Parameters.

Running an Application GUI

For software with a graphical interface (e.g. Octave, Blender) that is not yet in the Interactive Apps list, use Trillium Desktop — a full remote desktop environment running in your browser.

💡
Example: Running Octave's graphical interface inside a Trillium Desktop session.
1
Launch Trillium Desktop

Go to Interactive Apps → Trillium Desktop. Choose cores, memory, and job length, then click Launch.

2
Connect to the Desktop

On the My Interactive Sessions page, optionally adjust Image Quality and Compression for your connection speed, then click Connect to Trillium Desktop.

3
Open a terminal and load modules

Once the desktop loads, open a terminal within it and load the required environment modules:

$ module load StdEnv/2023 gcc/12.3 openmpi/4.1.5 octave/7.2.0
4
Launch the GUI application 
$ octave --gui

You can also inspect available binaries for any module:

$ ls $EBROOTOCTAVE/bin

Terminal Access

Open OnDemand provides a web-based terminal that works just like a regular SSH session — no local client required.

Access it via: Clusters → Trillium Shell Access. A new browser tab opens with a full terminal session on Trillium's login node.

Debugging Errors

If an interactive application fails to start or crashes, check the session logs:

1
Go to My Interactive Sessions

Find the session that failed (it will show an error status).

2
Click the output.log link

This opens the session's standard output and error log directly in the browser.

3
Note your Session ID

It's the long string like 8feb45fa-bc65-4846-8398-2a73c1bf8e5a shown on the session card.

Session Card — Output/Error Log Link
Fig. 13 — Session card: the Output/Error Log link (highlighted) and Session ID are key for debugging
📋
When submitting a support ticket to SciNet, include:
  • The output.log file contents
  • Your Session ID
  • Any other relevant context (modules loaded, job parameters, etc.)
You can also use the button directly on the session card to open a pre-filled ticket. Or contact: support@scinet.utoronto.ca

Porting Python Environments

⚠️
Important: Due to the switch from CentOS 7 to Rocky Linux 9 and the updated software stack, existing virtual environment Jupyter kernels may not work right away in the OnDemand portal.

To fix an existing Python virtual environment kernel:

1
Open a terminal on Trillium

Use Clusters → Trillium Shell Access or the >_ button on an active session.

2
Load required Trillium modules and activate your virtual environment 
$ module load StdEnv/2023 python/3.11
$ source ~/my_venv/bin/activate
3
Run venv2jup 
$ venv2jup

This registers your environment as a Jupyter kernel compatible with the new OS.
